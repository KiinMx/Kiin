Nombre del proceso: 

**Proceso de Establecimiento de Requisitos para la Elaboración de la Modificación**

**Descripción:**

Este proceso se enfoca en transformar la necesidad del cliente y el análisis previo del sistema en especificaciones técnicas claras y aceptadas antes de proponer soluciones.

|  |  |  |  |  | Para asegurar que el requisito sea **comprobable**, llenar obligatoriamente la columna *"Criterio de Aceptación"*. Un criterio de aceptación debe responder a la pregunta: *¿Cómo comprobar físicamente que esto ya funciona?* |
|  |  |  |  |  | Llenar el apartado 3 “Listado de tareas técnicas”, vinculando cada ID de requisito (RF/RFN) con sus tareas de implementación. Para decidir *cómo* se debe realizar la tarea, se debe consultar el **WMC (Weighted Methods per Class)** en el documento **AT\_\[ID\_MR\]**. **1\. ¿Dónde encontrar el WMC?** Abrir el documento **AT** del MR actual. Ir a la **Sección B1: Clases y componentes afectados**. Localizar la columna **"Estado actual"**. El número que aparece ahí es el WMC actual del archivo. Dependiendo del valor del WMC, la tarea técnica en el DER debe redactarse siguiendo estas reglas: 0 a 10 Bajo  Se permite modificar directamente el componente o clase. 11 a 15 Medio Evaluar si el cambio es pequeño. Si es complejo, preferir una función externa. Más de 15 Alto **PROHIBIDO** añadir más lógica directa. Se debe crear una nueva función, hook o clase.  |
